# PiNNTORP API Reference (Iteration 3)

The PiNNTORP backend provides a unified RESTful API and WebSocket interface on **Port 5500**.

## 1. Authentication
### `POST /login`
Provides authentication services through register, login, and logout. All protected routes require a **session ID** generated by the server through authentication.

**Request**:
```json
{
    "action": "register" / "login" / "logout",
    "username": "...",
    "password": "..."
}
```
- Action:
- - "register": Creates a new user account 'username' with 'password' as the password. Also starts session, generating session ID for new user and returning it alongside player ID.
- - "login": Attempts to sign user into the server by first finding whether a user with name 'username' exists, then checks password. Starts session on successful login, generating session ID and returning it alongside player ID.
- - "logout": Ends user session, invalidating session ID.
- Username:
- - The username of the user account.
- Password:
- - The password of the user account.

**Response**:
```json
{
    "sessionID": "...",
    "playerID": 1234,
    "success": true / false
}
```
- Session ID:
- - The unique identifier that represents this login session, acting as an identification and authorization token for this API.
- Player ID:
- - Returned on login and register. The integer player identifier is the user's identifier.
- Success:
- - Indicates the result of the action request, set to true if there were no issues.

## 2. Users
### `POST /user`
Provides user account balance actions.  
*NOTE: this endpoint will most likely be used for getting and searching user profiles.*

**Request**:
```json
{
  "sessionID": "...",
  "action": "deposit" / "withdraw" / "getBalance",
  "amount": 12.34
}
```
- Session ID:
- - The session ID of the user requesting this action.
- Action:
- - "deposit": Adds the amount to the user's balance.
- - "withdraw": Takes the amount from the user's balance, and indicates success.
- - "getBalance": Gets the user's current balance.
- Amount:
- - The amount of money to add or remove to or from a user's balance.

**Response**:
```json
{
  "balance": 12.34,
  "success": true / false
}
```
- Balance:
- - The user's account balance after the action was performed.
- Success:
- - Indicates whether the action was performed successfully.

### `POST /friends`
Provides user friends list and friend request actions.

**Request**:
```json
{
  "sessionID": "...",
  "action": "add" / "remove" / "list-friends" / "accept" / "decline" / "cancel" / "list-requests" / "find",
  "playerID": 1234,
  "friendID": 1234,
  "targetUsername": "..."
}
```
- Session ID:
- - The session ID of the user requesting this action.
- Action:
- - "add": Initiate the process of adding a user as a friend by sending them a friend request.
- - "remove": Remove a friend from the user's friends list.
- - "list-friends": Returns the user's friends list.
- - "accept": Accepts a friend request, adding the sender to the user's friends list.
- - "decline": Declines a friend request, cancelling the friend request for the sender.
- - "cancel": Cancel a sent friend request, removing it from the recipients friend request list.
- - "list-requests": Returns the user's sent and received friend requests.
- - "find": Find a user by username.
- Player ID:
- - The player ID of the sender.
- Friend ID:
- - The player ID of the recipient.
- Target Username:
- - The username to search for in the list of users.

**Response**:  
*NOTE: only applies to actions "list-friends", "list-requests", and "find", all other actions return HTTP status code 200 on success with no response body.

"list-friends": 
```json
{
  "friends": [
    {
      "playerID": 1234,
      "username": "..."
    },
    ...
  ]
}
```
- Friends:
- - An array representing a user's friends list.

"list-requests":
```json
{
  "sent": [
    {
      "playerID": 1234,
      "username": "..."
    },
    ...
  ],
  "received": [
    {
      "playerID": 1234,
      "username": "..."
    },
    ...
  ]
}
```
- Sent:
- - An array representing a user's sent friend requests.
- Received:
- - An array representing a user's received friend requests.

"find":
```json
{
  "found": true / false,
  "playerID": 1234,
  "username": "..."
}
```
- Found:
- - Whether or not a user with a matching username was found.
- Player ID:
- - The player ID of the found player. This property is only present if found is true.
- Username:
- - The username of the found player.

## 3. Lobbies
### `POST /lobby`
Provides lobby actions.

**Request**:
```json
{
  "sessionID": "...",
  "action": "create" / "list" / "join" / "leave",
  "lobbyName": "...",
  "lobbyID": "..."
}
```
- Session ID:
- - The session ID of the user requesting this action.
- Action:
- - "create": Create a lobby with the provided lobby name.
- - "list": Returns a list of currently open lobbies.
- - "join": Join the lobby by lobby ID
- - "leave": Leave the lobby the user is currently in.
- Lobby Name:
- - The name to call the lobby.
- Lobby ID:
- - The lobby ID of the lobby to join.

## 4. WebSockets

Connect to `ws://localhost:5555/` to participate in real-time features.

### Message Formats
- **Join Lobby**: `{"type": "join_lobby", "lobbyId": "..."}`
- **Game Move**: `{"type": "move", "action": "hit"}`
- **Broadcast**: Server sends `{"type": "update", "state": {...}}` to all clients in the lobby.
